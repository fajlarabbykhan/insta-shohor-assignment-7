Questions And Answers

JavaScript is a single-threaded programming language that is interpreted by the computer. This means the JavaScript source code does not need to be compiled before being sent to the browser. The raw JavaScript code can be passed to an interpreter, which will run it for us. Unlike other language, JavaScript is a dynamically typed language. This means that variables declared with var can store any data type, including int, text, boolean, and complex data types such as object and array. JavaScript's slowness is due to its absence of a type system. Because of the information it knows about the data, such as its type and size, a statically typed language can write more efficient machine code.

The event loop is the key of asynchronous programming in JavaScript. All actions in JS are performed on a single thread, but thanks to a few clever data structures, we get the illusion of multi-threading. Let's have a look at what goes on behind the scenes. The call stack is in charge of keeping track of all the actions in the queue that need to be completed. When a function completes, it is removed from the stack. The event queue is in charge of dispatching new functions to the stack for execution. It maintains the correct sequence in which all operations should be sent for execution by using the queue data structure. An async function is submitted to a browser API whenever it is called. These are built-in APIs in the browser. The API initiates its own single-threaded activity based on the command received from the call stack. The setTimeout method is an example of this. When a setTimeout operation is completed in the stack, it is forwarded to the associated API, which waits until the specified time has passed before returning the operation to be executed. What address does it use to send the operation? The queue for events. As a result, we have a cyclic system for conducting async JavaScript tasks. Although the language is single-threaded, the browser APIs run in their own thread. The event loop aids this process by checking if the call stack is empty on a regular basis. New functions are added from the event queue if it is empty. If it isn't, the current function call will be handled.

Local Storage: The data is stored in the localStorage with no expiration date. When the browser is closed, the data is not destroyed, and it will be available the next day, week, or year.
Session Storage: The sessionStorage is similar to the localStorage object, with the exception that it only saves data for one session. When the user exits the specified browser tab, the data is destroyed.
